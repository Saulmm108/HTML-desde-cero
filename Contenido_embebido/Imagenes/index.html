IMAGENES


      El peso de una imagen para sitios web entre menos peso mejora la velociedad de carga de un sitio web
      podria estar entre unos 20 o 30 KB.

      Ademas al realizar una pagina web es preferible tener las imagenes, git, etc en un archivo dentro del
      proyecto asi no dependes de otra pagina web que puede quitar la imagen o cerrar el sitio web de donde 
      sacas la imagen.

      Existen 2 tipos de imagenes:
        
      1. Vectoriales: Son matematicos (se crean con formulas)
      y no tienen resolucion, es decir, no se pixelean
      2. De mapa de bits: Son muchos cuadrados de colores (pixeles)
      que juntos forman una imagen. A más pixeles más calidad de la
      imagen

      Formatos de Imagenes.
      
      -svg: vectorial.

      
      -jpg, jpeg: es un mapa de bits utilizado para 
      fotografias.

      
      -png: se utilizan para transparencia de imagenes.

      
      -gif: se utilizan para animaciones de una duracion
      de menos de 10 seg.

      
      -webp: una combinacion entre jpg y png.

      
      -avif: mejor compresion que webp.

      
      -heif: es el formato de apple pero no se recomienda para
      sitios web.

PICTURE

      Es una etiqueta para agregar varias imagenes a la vez
      esto es para asegurarse de que las imagenes carguen en 
      todos los navegadores.

      Entonces puedes tener una imagen con varios formatos 
      de imagen jgp, png, etc. Esto es para que la misma
      imagen funcione en todos los navegadores.
